# 饮食情况追踪与 AI 分析建议 — 技术栈说明

本文档按模块详细说明项目各部分使用的技术栈、版本与职责。

---

## 一、整体架构

项目采用**前后端分离**：

- **后端**：Node.js + Express，提供 REST API，端口默认 3000。
- **前端**：独立 SPA（Vite + React），开发端口 5173，通过 Vite 代理将 `/api`、`/health` 转发到后端。
- **数据**：MySQL 持久化；后端使用连接池访问，迁移脚本基于原生 `mysql2` 执行 SQL 文件。

```
┌─────────────────┐     HTTP /api/*      ┌─────────────────┐     MySQL      ┌──────────┐
│  前端 (Vite)    │ ───────────────────► │  后端 (Express)  │ ──────────────► │  MySQL   │
│  React + ECharts│   proxy → :3000      │  Node.js        │   mysql2       │  diet_   │
│  :5173          │                     │  :3000          │                 │  tracking│
└─────────────────┘                     └─────────────────┘                 └──────────┘
                                                 │
                                                 │ 可选：OpenAI API（AI 报告）
                                                 ▼
                                        ┌─────────────────┐
                                        │  OpenAI API     │
                                        │  (gpt-4o-mini等) │
                                        └─────────────────┘
```

## 二、后端技术栈

### 2.1 运行环境与语言

| 项目 | 技术 | 说明 |
|------|------|------|
| 运行时 | **Node.js** | 建议 18+，CommonJS 模块（require） |
| 语言 | **JavaScript** | 注释采用 JSDoc，便于类型提示与文档 |

### 2.2 核心依赖（package.json）

| 依赖 | 版本 | 用途 |
|------|------|------|
| **express** | ^4.18.2 | Web 框架：路由、中间件、JSON 解析（express.json()） |
| **mysql2** | ^3.6.5 | MySQL 驱动，使用 **Promise 接口**（mysql2/promise），连接池访问 |
| **dotenv** | ^16.3.1 | 加载 `.env` 环境变量（DB_*、PORT、OPENAI_*、CRON_*） |
| **node-cron** | ^3.0.3 | 定时任务：每日 0 点（Asia/Shanghai）执行饮食汇总 |
| **openai** | ^4.52.0 | 官方 SDK，调用 OpenAI 生成日报/周报；未配置 KEY 时使用规则引擎降级 |

### 2.3 后端目录与职责

| 路径 | 技术/职责 |
|------|------------|
| **backend/server.js** | Express 应用入口：注册所有 API 路由、监听 PORT |
| **backend/config/db.js** | 使用 **mysql2 连接池**，对外提供 `query(sql, params)`，UTF-8（utf8mb4） |
| **backend/services/*.js** | 业务逻辑层：无框架，纯 Node 模块，通过 require 按需加载 |
| **backend/jobs/dietSummaryCron.js** | **node-cron** 定时任务，调用 dietSummaryService 做每日汇总 |

### 2.4 服务层技术要点

| 服务 | 涉及技术 | 说明 |
|------|----------|------|
| dietSummaryService | mysql2、日期处理 | 从 Meal_Records 聚合到 Daily_Diet_Summary |
| chartsService | mysql2、聚合查询 | 为 5 类图表提供 dates/series（日趋势、周对比、浪费率、速度、营养饼图） |
| promptService | 字符串模板 | 日报/周报 Prompt 模板与 generatePrompt |
| diagnosisService | 规则引擎 | 浪费率、摄入量、用餐速度等规则诊断，无 AI |
| aiAnalysisService | **openai**、JSON 解析、mysql2 | 调用 OpenAI 生成报告并写入 AI_Analysis_Reports；失败或未配置时降级规则报告 |
| recommendationService | mysql2 | 基于最近报告与诊断生成个性化建议列表 |
| deviceBindingService | mysql2 | 设备与用户绑定 CRUD，供遥测解析 user_id |
| telemetryService | 内存状态机、mysql2 | IDLE→SERVING→EATING→IDLE，写 Lunchbox_Meals、Meal_Curve_Data，结算时写 Meal_Records |
| mealsService | mysql2、游标分页、降采样 | GET /meals 游标分页；GET /meals/:id 详情；GET /meals/:id/trajectory 全量/增量/降采样 |

### 2.5 开发与测试

| 项目 | 技术 | 说明 |
|------|------|------|
| 单元测试 | **Node.js 内置 test runner**（node --test） | 测试 dietSummary、prompt、diagnosis、aiAnalysis 等服务 |
| 开发热重载 | **nodemon**（devDependencies） | 可选，未在 npm scripts 中默认配置，可自行 `nodemon backend/server.js` |

## 四、数据库与迁移

### 4.1 数据库

| 项目 | 技术 | 说明 |
|------|------|------|
| 数据库 | **MySQL** | 5.7 或 8.x，库名由 .env DB_NAME 指定（如 diet_tracking） |
| 字符集 | **utf8mb4** | 后端连接与表均使用 utf8mb4 |
| 驱动 | **mysql2** | 后端连接池 + 迁移脚本均使用 mysql2（Promise API） |

### 4.2 迁移机制

| 项目 | 技术 | 说明 |
|------|------|------|
| 执行方式 | **Node 脚本**（database/runMigrations.js） | 非迁移框架，直接读 `database/migrations/*.sql` 按文件名排序执行 |
| SQL 文件 | 纯 SQL | 001～007：Meal_Records、Daily_Diet_Summary、AI_Analysis_Reports、Device_User_Binding、Lunchbox_Meals、Meal_Curve_Data、Meal_Records 增加 lunchbox_meal_id |
| 执行命令 | `npm run migrate` | 需在项目根目录执行，且已配置 .env 中 DB_* |

### 4.3 主要表与用途

| 表名 | 用途 |
|------|------|
| Meal_Records | 单次用餐记录（含智能餐盒同步记录） |
| Daily_Diet_Summary | 按用户、日期汇总，供图表与 AI 报告 |
| AI_Analysis_Reports | 日报/周报内容与类型 |
| Device_User_Binding | 设备与用户绑定 |
| Lunchbox_Meals | 餐盒单次用餐主表（状态机结算写入） |
| Meal_Curve_Data | 餐盒就餐过程重量时序，供 trajectory 接口 |
