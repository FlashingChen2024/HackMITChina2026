# 服务器上 150 份用餐数据不显示时的排查

## 一、数据在哪儿显示？

| 数据来源 | 显示位置 | 说明 |
|----------|----------|------|
| **Meal_Records**（批量导入 / 遥测结算写入） | **统计图表** | 图表会先读 Daily_Diet_Summary，若无则按日聚合 Meal_Records，需 **user_id 一致** 且日期在范围内 |
| **Lunchbox_Meals**（仅遥测写入） | **用餐记录** 页、GET /api/v1/meals | 仅来自智能餐盒遥测的餐次会出现在「用餐记录」；Meal_Records 不会出现在该列表 |

所以：若 150 条在 **Meal_Records**，只会在**统计图表**里出现（按日聚合）；若在 **Lunchbox_Meals**，会在**用餐记录**页和图表相关逻辑里出现。

---

## 二、不显示的常见原因

### 1. 当前登录用户与数据所属用户不一致

图表和用餐记录都按 **当前登录用户（JWT 里的 userId）** 过滤。若 150 条记录的 `user_id` 不是当前账号（例如是 2 或 0），就不会显示。

**处理**：用数据所属账号登录，或把该批数据的 `user_id` 改为当前用户（需在库中更新）。

### 2. 前端连的不是「有 150 条数据」的那台服务器

若 150 条在**生产库**，而你在本地浏览器里访问的是 **localhost:8080**（本地后端），前端只会看到本地库的数据。

**处理**：让前端请求生产 API。在 `frontend/.env` 或 `.env.production` 中设置：
```bash
VITE_API_BASE=https://api.mit.chenyuxia.com
```
然后重新构建或重启前端，再登录、看图表。

### 3. 日期范围没包含那 150 条的日期

在统计图表里选好**开始日期**和**结束日期**，要点「加载」才会请求该区间。若 150 条不在你选的区间内，就不会出现。

**处理**：把日期范围拉大到包含所有有数据的日期，再点「加载」。

### 4. 图表只读 Daily_Diet_Summary 时未做汇总（已支持回退）

图表逻辑会先查 **Daily_Diet_Summary**，没有则从 **Meal_Records** 按日聚合，所以通常**不跑汇总也能看到** Meal_Records。若你希望汇总表也有数据，可对这批日期做一次「按范围汇总」。

---

## 三、对「有 150 条的那段日期」做批量汇总

若 150 条都在 **Meal_Records**，且希望写入 **Daily_Diet_Summary**（方便报表等用汇总表），可对这段日期批量跑汇总：

```bash
# 示例：对 2025-01-01 ～ 2025-12-31 每天执行汇总（需先登录拿到 Token）
curl -X POST http://127.0.0.1:8080/api/diet/summary/run_range \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <你的Token>" \
  -d '{"start_date":"2025-01-01","end_date":"2025-12-31"}'
```

把 `start_date` / `end_date` 换成你 150 条数据实际所在的日期范围；生产环境把 `http://127.0.0.1:8080` 换成生产 API 地址。

---

## 四、自检清单

1. 150 条是在 **Meal_Records** 还是 **Lunchbox_Meals**？→ 决定在「统计图表」还是「用餐记录」里看。
2. 这 150 条的 **user_id** 是否与当前登录用户一致？→ 不一致则换账号或改数据。
3. 前端请求的 **BASE** 是否指向「存有这 150 条」的那台后端？→ 本地数据用 localhost，生产数据用生产 API。
4. 统计图表的**日期范围**是否包含这 150 条的日期？→ 选对范围并点「加载」。
