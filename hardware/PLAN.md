核心要求**：硬件开发者必须严格按照 M1 至 M5 的顺序进行开发。**每一个 Milestone 必须完成相应的“验收标准”测试后，才允许进入下一阶段的代码编写。**

### 🟢 M1: 阵列点亮与工厂标定 (Bring-up & Calibration)（已完成）

- **开发任务**:
  
  1. 引入 HX711 驱动库，初始化 4 个传感器实例。
    
  2. 实现“工厂标定模式”宏定义开关及对应的纯串口打印逻辑。
    
  3. 在标定模式下，使用标准砝码测算出 4 个通道的比例因子。
    
  4. 实现“生产模式”，将算出的因子硬编码写入，输出真实克数。
    
- **脱机验收标准 (Serial Monitor)**:
  
  1. 开启标定模式：串口高频打印原始 ADC 极大数值。
    
  2. 切换为生产模式：放上 100g 砝码，串口准确打印 `Grid X: 100.0g`。同时按压 4 个角，各路数据独立变化且无通道串扰。
    

### 🟢 M2: 网络基建与时钟同步 (Network & NTP)（已完成）

- **开发任务**:
  
  1. 引入 WiFi、HTTPClient 及 TLS 相关基础库。
    
  2. 实现连接指定 SSID/Password 的逻辑。
    
  3. 实现 NTP 时间服务器连接与本地 Unix 时间获取。
    
  4. 针对 `https://api.mit.chenyuxia.com/api/v1/ping` 发起 GET 请求（需配置跳过证书校验）。
    
- **脱机验收标准 (Serial Monitor)**:
  
  1. 串口打印分配到的内网 IP 地址。
    
  2. 串口成功打印长度为 10 位的 Unix 时间戳。
    
  3. 成功打印后端探活接口返回的 `HTTP 200` 状态码及 `{"status":"pong"}` JSON 响应体。
    

### 🟡 M3: 核心业务组装与上报 (Data Ingestion)（已完成）

- **开发任务**:
  
  1. 引入 JSON 序列化库 (如 ArduinoJson)。
    
  2. 读取硬件 MAC 地址。
    
  3. 按照 `API.md` 的要求，将 M1 的真实重量数据、M2 的时间戳和 MAC 地址组装为 JSON 字符串。
    
  4. 每隔 5 秒向 `/api/v1/hardware/telemetry` 端点发起 POST 请求。
    
- **双端联调验收标准**:
  
  1. ESP32 串口打印出构建完毕的完整 JSON 字符串，且 HTTP 状态码返回 200。
    
  2. **跨端确认**：后端开发者在服务器日志中，确切看到每 5 秒准时收到一条格式完全合法的数据流。
    

### 🟡 M4: 异步重构与边缘滤波 (Async & Filtering)（已完成）

- **开发任务**:
  
  1. 彻底清查并移除代码中的 `delay()` 语句（尤其是发包后的强制等待）。
    
  2. 实现 `millis()` 时间差分发器：确保传感器读取维持在 10Hz，网络发包维持在 0.2Hz (5000ms)。
    
  3. 植入“去极值均值滤波”算法至传感器读取逻辑中。
    
- **验收标准**:
  
  1. 串口输出观测：传感器数据的打印在高速刷屏，但 HTTP POST 的日志严格每隔 5 秒才出现一次。
    
  2. **抗扰测试**：在秤盘上放置物品，并在网络发送前 1 秒内，用手指快速弹击秤盘。后端接收到的该次数据，其重量波动不得超过 $\pm 3g$。
    

### 🟠 M5: 工业级自愈机制 (Robustness)（已完成）

- **开发任务**:
  
  1. **断线重连**：在主循环检测 WiFi 状态。若断开，执行非阻塞的重连逻辑，期间停止发起 HTTP 请求以防 Watchdog 复位。
    
  2. **超时熔断**：设置 HTTP 请求的 Timeout 上限（建议 3000ms），防止因服务器无响应导致 ESP32 线程挂死。
    
- **验收标准**:
  
  1. 系统正常运行期间，人为关闭测试热点。ESP32 串口应打印 `WiFi Disconnected`，但程序不应 Crash 或重启。
    
  2. 恢复测试热点，ESP32 必须在 15 秒内自动重新获取 IP，并无缝恢复向后端的 HTTP POST 上报。
