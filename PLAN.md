# 🚀 交付物三：终极收官敏捷落地计划 (PLAN v4.3)

**实施原则**：冻结新需求，集中火力跑通 GORM 联表查询与全链路联调。

### 🟢 M1: 社区双向查询机制落地（已完成）

- **任务**：
  
  1. 在原有的 `communities` 模块控制器中增加 `GET /communities` 路由。
    
  2. 使用 `GORM` 联表 `user_communities` 结构，获取当前用户的圈子列表及人数聚合。
    
- **验收**：创建一个“减脂圈”和一个“增肌圈”，用 A 账号加入减脂圈，用 B 账号加入两个圈。测试 `GET /communities`，A 只能刷出 1 条记录，B 能刷出 2 条记录。
  

### 🟡 M2: 全链路数据造影 (E2E Testing)（已完成）

- **任务**：
  
  1. 前端通过扫码调用 `POST /devices/bind` 绑定硬件。
    
  2. 硬件兄弟开启 ESP32 开始灌入一顿包含 4 个格子的物理曲线。
    
  3. 吃饭结束后，前端调用 `PUT /meals/{meal_id}/foods` 上传 4 个菜品的名称和卡路里。
    
  4. 前端调用 `GET /users/me/ai-advice` 成功获取“吐槽式点评”。
    
  5. 前端调用 `GET /communities/{id}/dashboard`，在社区大屏上看到这顿饭拉高了全社区某道菜的“平均打饭量”。
